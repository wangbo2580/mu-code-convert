# /dev-project 命令规范

> 版本: 1.0.0
> 创建日期: 2026-01-14

---

## 命令说明

**命令名称**: `/dev-project`

**用途**: 基于指定目录下的规划文档和需求文档，自动完成项目的任务拆解、进度跟踪和完整开发。

---

## 使用方式

```
/dev-project <文档目录路径>
```

**示例**:
```
/dev-project @MuCodeConvert\docs\
```

---

## 执行流程

### 阶段一：文档分析
1. 读取指定目录下的所有文档（MVP规划、需求文档等）
2. 理解项目背景、功能需求、技术栈、验收标准

### 阶段二：任务拆解
1. 根据文档内容拆解任务
2. 按优先级排序（P0 > P1 > P2）
3. 输出任务拆解文档到 `docs/任务拆解_{项目名}.md`

### 阶段三：进度追踪
1. 创建任务开发进度表 `docs/开发进度表_{项目名}.md`
2. 每完成一个任务，实时更新进度表
3. 记录开始时间、完成时间、状态、备注

### 阶段四：逐步开发
1. 按任务优先级和依赖关系顺序开发
2. 严格按照文档中的技术规范实现
3. 每个功能完成后更新进度表
4. 代码完成后不自动启动，由用户手动验证

### 阶段五：完成确认
1. 所有任务完成后，更新进度表为"已完成"
2. 输出完成总结

---

## 输出文档

| 文档 | 路径 | 说明 |
|:--|:--|:--|
| 任务拆解文档 | `docs/任务拆解_{项目名}.md` | 详细的任务分解 |
| 开发进度表 | `docs/开发进度表_{项目名}.md` | 实时更新的进度 |

---

## 注意事项

1. **严格遵循文档**: 所有实现必须符合需求文档中的规范
2. **不自动启动**: 编码完成后不启动 dev server
3. **中文回复**: 全程使用中文沟通
4. **实时更新**: 每完成一个任务立即更新进度表
5. **完整实现**: 确保所有功能完整实现，不遗漏

---

## 调用示例

用户输入:
```
/dev-project @NewProject\docs\
```

Claude 将:
1. 读取 `NewProject\docs\` 下的所有规划和需求文档
2. 创建任务拆解文档
3. 创建开发进度表
4. 按顺序完成所有开发任务
5. 实时更新进度表

---

> 此命令适用于任何有完整规划文档和需求文档的项目
